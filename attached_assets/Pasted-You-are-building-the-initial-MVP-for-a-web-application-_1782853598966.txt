You are building the initial MVP for a web application called **PWR Risk Review Coordinator**.

This application will replace an existing Microsoft Forms + Power Automate + Excel tracker workflow used to intake, coordinate, schedule, and track PWR risk review meetings.

The first version should be a working MVP that runs in Replit, but the codebase must be designed so it can later be exported and deployed as a **Databricks App**.

Do not build this as a throwaway Replit prototype. Build it as a clean, portable, production-minded repo.

---

# 1. Primary Goal

Build a web app that allows a coordinator to:

1. Create and review risk review scheduling requests.
2. Track all active and completed risk review requests in a dashboard.
3. Classify each request as Major or Non-Major.
4. Classify the business line.
5. Track Pre-Risk, Formal Risk, and Final Risk workflow stages.
6. Generate editable email drafts.
7. Generate calendar invite preview data.
8. Track meeting dates, target dates, leads, statuses, next actions, notes, and reschedules.
9. Maintain configurable business rules and email templates.
10. Prepare the codebase for later Outlook/Graph and Databricks integration.

The MVP should not send real emails or create real Outlook calendar events yet. It should generate previews/drafts only.

Create a backend rules service in:

```text
src/server/rules/

Major Opportunity Rule

A request is Major if the selected risk triggers include either:

Trigger 1
Trigger 2

Otherwise it is Non-Major.

Business Line Classification

The request form allows multiple business lines.

Supported business lines:

BESS
Decarbonization
GHI
Nuclear
Solar
Mining

Classification logic:

If BESS and Solar are both selected, classify as BESS + Solar.
If BESS is selected and Solar is not selected, classify as BESS.
If Solar is selected and BESS is not selected, classify as Solar.
If GHI is selected and neither BESS nor Solar is selected, classify as GHI.
Otherwise classify as Other.
EPC Prime Rule

If is_epc_prime = true, collect and display the EPC-prime attendee roles, including:

Business-Line Director

Contract Review Request RVW #
Attorney
Regional GP Manager
Regional Risk Manager
Other Attendees

If is_epc_prime = false, collect and display the standard attendee roles.

Validation Warnings

On the request detail page, show warnings when:

Attorney is missing.
Contract Review Request RVW # is missing.
Formal Risk is requested but legal information is incomplete.
Final Risk is requested but Formal Risk Discussion Date is missing.
Risk Identification Meeting status is No or blank.
Pre-Risk target date is after Formal Risk target date.
Proposal due date is before Formal Risk target date.
Required attendees are missing.
Request has no risk triggers selected.
Request has no business line selected.

Warnings should not block saving in the MVP. They should be visible to the coordinator.
